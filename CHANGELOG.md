# CHANGELOG

本文档根据 `debug.md` 与 `阿里云OSS部署-Codex执行文档` 整理，用于记录个人照片展示网站上线过程中的主要功能更新、部署调整与 Bug 修复情况。



---

## [Unreleased] - 个人照片展示网站上线阶段

### 新增功能

#### 前台展示

- 新增 Photo 层级的 CLIP + TSP 智能排序能力，用于优化同一地点相册内照片的视觉过渡顺序。
- 新增照片展示区域优化：扩大 Photo 层级液态玻璃展示框内的照片可视区域，减少上下左右留白，以便一次显示更多照片。
- 新增背景图片轮播功能：前台页面背景由固定单张图片改为从背景图片集中按顺序展示。
- 新增背景切换功能：在前台返回按钮下方添加“切换背景”按钮，可手动切换到下一张背景图。
- 新增 Years 层级背景切换按钮显示逻辑：Years 页面不显示返回按钮，但保留背景切换按钮。
- 新增背景过渡方案设计：针对背景切换过于生硬的问题，考虑柔和过渡效果。
- 新增背景铺满全屏方案：采用裁切方式保证不同显示器比例下背景图都能铺满页面。
- 新增 Photo 层级透明滚动条样式，使照片浏览区域视觉更干净。

#### 后台管理

- 新增后台批量导入照片功能，支持一次选择多张照片。
- 新增批量导入文件列表展示：选择多张照片后，显示第一张照片名称及总数量，并可展开查看全部文件名。
- 新增照片文件名解析能力：根据 `序号_地点_日期` 格式自动解析地点与日期。
- 新增“解析日期地点”按钮：点击后解析第一张照片文件名，并自动填充拍摄日期与地点字段。
- 新增后台“重新排列相册”功能：支持按“时间-地点”选择某个相册并重新排列该相册内照片。
- 新增“重新排列全部相册”功能，并加入二次确认机制。
- 新增管理入口按钮点击反馈效果，提升后台操作的交互反馈。
- 新增修改照片信息的三级递进选择菜单：Years → Locations → Photos，便于快速定位目标照片。
- 新增修改照片信息时的缩略图预览功能，当前选中照片会在设置框右下角显示缩略图。
- 新增后台背景管理入口，按钮命名为“修改背景”。
- 新增背景图片批量上传功能。
- 新增背景播放顺序调整功能。
- 新增背景播放间隔设置功能，支持 3 分钟、5 分钟、10 分钟。
- 新增后台“下一张背景”功能，可在后台手动切换背景。
- 新增背景图片删除功能：每张背景图后方增加删除按钮，并加入二次确认机制。
- 新增背景遮罩透明度调节功能，支持通过拖动条调整前台黑色半透明遮罩强度，并加入二次确认机制。
- 新增批量修改地点功能：可选择某一年某一地点下的全部照片，并统一修改为新的地点名称。

#### 云端部署与 OSS 支持

- 新增阿里云 OSS 作为照片资源存储方案，用于存储照片原图、缩略图、大图和背景图。
- 新增 OSS 推荐目录结构：
  - `photos/original/`：原图
  - `photos/thumb/`：缩略图
  - `photos/large/`：大图
  - `backgrounds/`：背景图
  - `admin-temp/`：临时上传目录
- 新增 OSS 图片样式规划，支持 `thumb` 与 `large` 两类访问场景。
- 新增前端 OSS URL 工具函数规划，包括 `getOriginalUrl`、`getThumbUrl`、`getLargeUrl`。
- 新增公开环境变量 `VITE_OSS_PUBLIC_BASE_URL`，用于配置图片公共访问域名。
- 新增 `.env.example` 配置规划。
- 新增 Vercel SPA 部署 rewrite 配置规划，避免刷新页面后路由失效。
- 新增 GitHub 仓库清理规则，要求不再提交本地照片、背景图、构建产物、环境变量和临时目录。
- 新增本地背景与 OSS 背景对比脚本，可自动上传新增背景，并列出需要删除的背景。
- 新增本地预览与云端资源区分策略：本地预览时只从本地加载图片，避免消耗 OSS 流量。

- 修复 Photo 层级自动排列未生效的问题，使后台点击重新排列后能够重新排列当前 Location 下的所有照片。
- 修复部分竖屏照片无法完整显示的问题。
- 修复 Location 和 Photo 层级返回按钮位置错误问题，将返回按钮放置在液态玻璃框体左上角上方，避免遮挡照片。
- 修复返回按钮被液态玻璃框体盖住的问题，确保返回按钮始终位于最上层。
- 修复 Photo 层级标题信息强调顺序错误的问题，将主标题与副标题中的地点、时间显示位置对调。
- 修复层级菜单中出现不易察觉的黑色线条问题。
- 修复底部级别菜单颜色过深的问题，使其液态玻璃效果与顶部菜单保持一致。
- 修复浏览器缩放后上下菜单位置漂移的问题，使菜单相对于背景框体保持固定居中。
- 修复底部菜单字体过大的问题，使其与顶部菜单日期字号保持一致。
- 修复底部菜单位置过高的问题，使其垂直中线与背景框体下边缘重合。
- 修复 Photo 层级底部菜单液态玻璃透视异常问题，使其透视当前照片颜色，而不是页面底部背景颜色。
- 修复 Years 层级标题冗余问题，删除“按年份排列”副标题，并将 `Photo Library` 居中显示。
- 修复 Years 层级液态玻璃框体下方留白过多的问题，使框体大小与内部年份按钮自适应。
- 修复上传背景后前台仍显示旧背景的问题。
- 修复背景图未完整显示的问题，调整为更适合不同显示器比例的展示方案。
- 修复刷新页面时仍显示最早固定背景图片的问题。

#### 后台管理修复

- 修复后台初始进入时默认同时显示多个操作模块的问题，改为点击具体操作后仅显示对应模块。
- 修复批量添加照片时只能解析第一个文件但未自动填充字段的问题。
- 修复文件名解析规则错误问题，正确识别 `序号_地点_日期` 格式：
  - 第一个下划线之前为序号；
  - 第一个下划线与第二个下划线之间为地点；
  - 第二个下划线之后为日期；
  - 日期格式为 `yyyyMMdd`，例如 `20260408`。
- 修复“重新排列相册”下拉菜单排序问题，使其按时间排序。
- 修复修改照片信息表单中照片选择效率低的问题，由一级照片菜单改为三级递进菜单。
- 修复修改照片信息设置框中重复出现两个地点输入框的问题，删除多余地点框。
- 修复缩略图预览显示不完整的问题，根据照片分辨率和长宽比例进行适当缩放。
- 修复后台按钮布局问题，将“重新排列相册”按钮移动到管理入口中，并放置在“当前图库”按钮右侧。

#### 云端部署与工程化修复

- 修复前端继续从本地路径读取图片的问题，规划改为通过 OSS 公共 URL 加载图片资源。
- 修复列表页可能直接加载原图的问题，规划统一使用缩略图 URL，降低加载压力和流量成本。
- 修复 GitHub 仓库可能包含大量本地图片资源的问题，要求移除 `data/uploads/`、`data/backgrounds/` 等目录的 Git 追踪。
- 修复敏感密钥可能暴露在前端或 GitHub 仓库中的风险，明确禁止将阿里云 AccessKey 写入前端代码或 `VITE_` 环境变量。
- 修复 OSS 跨域访问风险，增加 CORS 配置规划，支持本地开发域名、正式域名和 Vercel 域名访问。
- 修复本地预览消耗 OSS 流量的问题，本地环境改为读取本地图片资源，避免不必要的外网下行流量费用。

## 后续待关注事项

- OSS 外网下行流量、CDN 流量和图片处理费用需要持续关注。
- 域名 `rsddp.top` 若在中国大陆稳定访问，需要完成实名认证、ICP备案、HTTPS 配置和可能的 CDN 接入。
- 后台上传功能后续应通过后端生成 OSS 上传签名，不能在前端暴露阿里云密钥。
- 删除照片时需要同步处理 OSS 文件删除、照片元数据删除和本地备份。
- 移动端性能、图片懒加载、LiquidGlass 低性能设备降级仍需进一步优化。
